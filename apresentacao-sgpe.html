Sistema de Gestão de Projetos e Equipes — uma aplicação desktop Java MVC com persistência SQLite, testes automatizados e controle de acesso por perfil.

Como coordenar projetos, equipes e tarefas garantindo que cada pessoa veja só o que lhe compete sem depender de um framework pesado?
Java 21 Swing SQLite MVC JUnit 5
Contexto

O cenário do projeto

Gerir projetos é coordenar pessoas, prazos e responsabilidades ao mesmo tempo.

→ Times crescem e as tarefas se multiplicam rapidamente.
→ Cada papel precisa enxergar apenas o que lhe cabe.
→ Organização e controle viram requisito, não luxo.
Problema

O problema a resolver

Dados dispersos

Informação espalhada em planilhas e anotações soltas.

Sem rastreabilidade

Responsabilidades difusas e tarefas sem dono claro.

Sem controle de acesso

Todos veem tudo, sem limite por perfil.

Desempenho invisível

Progresso do projeto difícil de medir e enxergar.

Resultado: retrabalho, ruído e decisões tomadas no escuro.
Solução

A solução: SGPE

Uma aplicação desktop Java que centraliza tudo num só lugar.

✓ Cadastro unificado de usuários, projetos, equipes e tarefas.
✓ Acesso por perfil embutido no próprio fluxo de uso.
✓ Roda local, sem servidor nem framework pesado.
Em uma frase

Software real, organizado por responsabilidades — não um protótipo descartável.

Centralizado Seguro Testado
